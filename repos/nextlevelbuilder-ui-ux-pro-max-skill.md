## What it is

Pixel-perfect UI/UX aesthetic rules for AI-driven frontend generation.

## Problems solved

- AI often generates 'ugly' or non-standard UI code by default.
- Divergence between professional design mockups and AI implementation.
- Lack of aesthetic grounding in general-purpose coding models.

## How to get started

/plugin install ui-ux-pro-max-skill # In compatible environments
# Or add instructions to your system prompt.

## Detailed report

### 1. Positioning
UI-UX Pro Max is a specialized knowledge base and 'Skill' for LLMs that forces them to follow elite design systems during code generation. It turns an AI from a basic coder into a senior UI/UX engineer.

### 2. Pain Points & Advantages
- **Pain Points Solved**: Dramatically reduces the time spent on manual CSS fixing after an AI generates a layout.
- **Differentiation**: Built on 'Master Rules' that cover spacing, typography, and accessibility (WCAG) natively in the instruction set.

### 3. Technical Architecture
- **Structure**: A hierarchical rule-set (Universal Rules -> Component Rules -> Interactive Rules).
- **Design Strategy**: Uses CSS-in-JS and Tailwind as the primary implementation targets for the rules.

### 4. Core Implementation
- **Concept**: Implements 'design tokens' as hard-coded constraints for the LLM's imagination.
- **Practice**: Demonstrates how to write 'negative instructions' to avoid common AI design tropes (like boring blue buttons or excessive rounded corners).

### 5. Code Organization
- `/rules`: The core MD files containing design logic.
- `/examples`: Before-and-after code snippets showing the impact of the skill.

### 6. Quick Start Suggestion
Read `rules/typography.md` to see how the skill forces modern, readable font hierarchies on any generated project.

### 7. Summary
**Essence**: A 'design soul' for AI coders. Key takeaway is the methodology of grounding creative tasks in rigorous technical constraints.
