## What it is

A radical 'Autonomous Company' framework that orchestrates agents to handle customer service, billing, and fulfillment with zero human intervention.

## Problems solved

- Extreme manual overhead in running small internet businesses (handling emails, checking stripe, updating shipping).
- Lack of a 'Vertical Orchestrator' that understands how to connect business strategy with operational actions.
- Scaling limits of human-based service companies.

## How to get started

git clone https://github.com/paperclipai/paperclip
# Set your Business-Intent and connect your service APIs.
npm run start:autonomous

## Detailed report

### 1. Positioning
Paperclip is a high-level framework designed to run a 'Zero-Human Business.' It orchestrates a fleet of specialized agents (Sales, Support, Ops) that coordinate through a central 'Business Brain' to handle everything from lead generation to customer refunds autonomously. Aimed at solo-founders and high-automation startups.

### 2. Pain Points & Advantages
- **Pain Points Solved**: Solves the 'Context Leak' between different departments—the Sales agent knows exactly what the Support agent told the customer, enabling a unified business presence.
- **Differentiation**: Features 'Strategic Drift Protection'—a high-level supervisor agent constantly monitors the sub-agents to ensure their actions align with the company's core mission and profitability.

### 3. Technical Architecture
- **Tech Stack**: Node.js/TypeScript, PostgreSQL (State), and multi-LLM orchestration.
- **Design Strategy**: Adopts an 'Actor-Network' model where business processes are defined as long-running stateful workflows.

### 4. Core Implementation
- **Logic Layer**: `lib/business-engine/planner.ts` translates high-level company goals into thousands of operational agent tasks.
- **Innovation**: Implements 'Automated P&L Tracking'—the agent fleet can actually stop itself if the cost of AI tokens exceeds the revenue generated by the task.

### 5. Code Organization
- `/departments`: Discrete logic for Sales, Support, and Logistics agents.
- `/governance`: Built-in constraints and safety rules for autonomous business decisions.

### 6. Quick Start Suggestion
Define a simple 'Digital Product Company' in the config and watch the agents autonomously setup a Stripe link, create a landing page, and start answering mock customer emails.

### 7. Summary
**Essence**: The 'ERP' for AI. Key taking is the management of multiple agents as a single, profitable business entity.
