# 需求

该项目目的是提供一个gitbook或者类似的页面，用来介绍一些stars数量高的项目

## 技术栈

- js相关技术栈，重点可以部署到gitbook上

## 页面功能介绍

- 需要将`/Users/xia_g/Desktop/github/RepoBrief-since-2025/github_repos.db`的数据生成对应的md文档，存储在`repos`目录下
- 页面需要展示的内容数据源为存储在`repos`目录下的md文档里的内容
- 页面的视觉稿为`/Users/xia_g/Desktop/github/RepoBrief-since-2025/doc/require.pen`
  - 页面有两套主题，一套是深色主题，一套是浅色主题，默认是深色主题
  - 页面的header模块左边是logo，自行发挥；右边是深色/浅色的切换按钮，以及对语言切换的按钮
  - 筛选模块里展示的筛选条件为`tags`,由于数据库里的`tags`是字符串，所以需要对字符串进行分隔，分隔符为逗号，分隔后的结果为`tags`的值，所以一个卡片可以有多个`tags`
  - 列表右侧有按照星星数多少的排序按钮，按钮旁边可以导出当前筛选条件下的所有数据，格式是excel
  - 列表的卡片展示的效果以数据源为准，项目名称旁边需要加一个链接图标，点击跳转到项目；点击卡片跳转到浮层，浮层里展示当前卡片的详细介绍，详见下一节

  ## 浮层功能介绍

  - 浮层的视觉稿为`/Users/xia_g/Desktop/github/RepoBrief-since-2025/doc/poplayer.pen`
  - 浮层里展示当前卡片的详细介绍，详细介绍以卡片对应md文档为准
